⚡ OpenAI Codex Code Mode
你
给我写一个 shell 脚本，搜索"电商 Banner"，自动下载前 10 条到 ./downloads 目录
🧑
⚡
Codex
我来帮你写这个批量下载脚本： #!/bin/bash
mkdir -p ./downloads
IDS=$(58pic search "电商 Banner" --json | jq -r '.[0:10][].id')
for id in $IDS; do
  URL=$(58pic download $id --json | jq -r '.url')
  curl -o "./downloads/$id.psd" "$URL"
  echo "✔ 下载完成: $id"
done 脚本已生成，保存为 download.sh 后运行即可。
💡
Codex 在代码生成场景最强，直接生成 shell / Python 脚本调用 58pic-cli，配合 --json 输出和 jq 实现完整自动化。

命令速查
全部可用命令

一键复制，开箱即用。所有命令支持 --help 查看详细参数。

命令列表
命令 说明 示例 类型
config init 交互式初始化配置（API Key 等） 58pic config init 配置
auth status 验证当前认证状态 58pic auth status 配置
search 关键词 / AI 向量搜索素材 58pic search "新年海报" --json 搜索
catalog 浏览素材分类目录 58pic catalog 搜索
models 列出平台可用的 AI 模型 58pic models 搜索
download <pid> 通过素材 ID 获取下载链接 58pic download 12345678 操作
api <route> 通用 API 透传（任意路由） 58pic api /v1/materials 操作
search --dry-run 预览请求内容，不实际执行 58pic search "X" --dry-run 调试

💡 全局参数： --json 输出 JSON  ·  --pretty 格式化 JSON  ·  --table 表格输出  ·  --api-key <key> 临时覆盖凭证

常见问题
FAQ

遇到问题？先看看这里。

如何获取千图网 API Key？ ▼
前往千图 AI 开放平台注册账号，在控制台「API 管理」页面创建密钥。获取后运行 58pic config init 填入即可。
安装时提示 "Permission denied"？ ▼
全局安装需要权限。推荐使用 nvm 管理 Node 版本：nvm install 20 && nvm use 20 && npm install -g github:58pic-open/cli
凭证优先级是怎样的？ ▼
从高到低：命令行 --api-key > 环境变量 58PIC_API_KEY > 配置文件 ~/.58pic/config
搜索没有结果怎么办？ ▼
1. 运行 58pic auth status 确认已认证
2. 换个关键词 / 使用中文
3. 加 --dry-run 检查请求是否正确
4. 在 GitHub Issues 提交反馈
GitHub 安装和 npm 安装有什么区别？ ▼
GitHub 安装（github:58pic-open/cli）始终对应最新提交，包含预编译 dist/ 无需构建；npm 安装对应已发布的稳定版本。开发环境推荐 GitHub 安装。
